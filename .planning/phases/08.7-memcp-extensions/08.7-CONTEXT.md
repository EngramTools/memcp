# Phase 08.7: memcp Extensions

## Goal

Extend the token-efficiency patterns from Phase 07.6 to `recall_memory` and add context-aware salience defaults that auto-adjust based on usage patterns.

## Context

Phase 07.6 added `fields` and `min_salience` to `search_memory`. This phase extends projection to `recall_memory` (same pattern, different tool) and explores context-aware defaults — if models consistently struggle to set good thresholds, the server can suggest or apply sensible defaults based on query patterns and result sizes.

## Depends on

Phase 07.6 (field projection and salience filtering proven on search_memory — DONE)

## Success Criteria

1. `recall_memory` supports the same `fields` param as `search_memory` — projected responses return only requested fields
2. If context-aware defaults are implemented: the server can suggest a min_salience threshold based on result distribution (e.g., "20 results found, 80% below 0.3 — consider min_salience: 0.3")

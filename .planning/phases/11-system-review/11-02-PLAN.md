  <action>
    Run `cargo clippy 2>&1` and fix the two largest warning categories:

    **Category 1: `empty_line_after_doc_comments` (~39 warnings)**
    These are `///` doc comments followed by an empty line before the item they document. The doc comment gets attached to the wrong item (usually a `use` import instead of the struct/function below).

    Fix strategy for each occurrence:
    - If the doc comment should be a MODULE-level doc: change `///` to `//!` (inner doc comment)
    - If the empty line is just formatting: remove the empty line so the doc comment attaches to the correct item
    - If the doc comment is above a `use` statement and describes the module: convert to `//!` inner doc comment
    - Common pattern in this codebase: module-level `///` comments at file top → convert to `//!`

    **Category 2: `needless_borrows_for_generic_args` (~20 warnings)**
    "The borrowed expression implements the required traits" — remove unnecessary `&` on arguments where the owned value already implements the trait.

    Fix: Remove the `&` prefix. Example: `some_fn(&string)` → `some_fn(string)` when `string: String` and the function accepts `impl AsRef<str>`.

    After each batch of fixes, run `cargo build` to verify compilation. Then run `cargo test --lib -- --test-threads=1` to verify no regressions.

    **IMPORTANT:** Do NOT change any logic or behavior. These are purely cosmetic/lint fixes. If a fix seems like it could change behavior, leave it with `#[allow(clippy::...)]` and a comment.
  </action>

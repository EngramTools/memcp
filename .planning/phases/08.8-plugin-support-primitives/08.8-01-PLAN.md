**Migration 018** — Create `crates/memcp-core/migrations/018_plugin_primitives.sql`:
```sql
ALTER TABLE memories ADD COLUMN IF NOT EXISTS event_time TIMESTAMPTZ;
ALTER TABLE memories ADD COLUMN IF NOT EXISTS event_time_precision TEXT
    CHECK (event_time_precision IN ('decade', 'year', 'month', 'day'));
ALTER TABLE memories ADD COLUMN IF NOT EXISTS workspace TEXT;
CREATE INDEX IF NOT EXISTS memories_workspace_idx ON memories (workspace)
    WHERE workspace IS NOT NULL;
```

Per CONTEXT.md: `event_time_precision` is TEXT with CHECK constraint (not Postgres ENUM — easier to extend per Pitfall 3 from RESEARCH.md). `workspace` is nullable TEXT — NULL means global. Partial index on workspace excludes NULLs for efficiency.

**Config structs** — Add three new config structs to `config.rs`:

1. `UserConfig`:
   - `birth_year: Option<u32>` — Required for resolving relative-age references ("when I was 6"). If not set, relative-age references stored without resolution.
   - `#[serde(default)]` on all fields.

2. `WorkspaceConfig`:
   - `default_workspace: Option<String>` — Default workspace applied when no CLI flag or env var set. NULL = global (no scoping).
   - `#[serde(default)]` on all fields.

3. `TemporalConfig`:
   - `llm_enabled: bool` — Whether to run LLM background worker for subtle temporal references. Default false.
   - Standard LLM provider fields if llm_enabled: `provider: String` (default "ollama"), `ollama_model: String`, `openai_model: String`, `openai_api_key: Option<String>`, `openai_base_url: Option<String>`.
   - `#[serde(default)]` on all fields.

4. Extend `RecallConfig` with three new fields:
   - `truncation_chars: usize` — default 200. Truncate recalled memory content to this many chars.
   - `preamble_override: Option<String>` — Custom preamble text for `--first`. None uses hardcoded default.
   - `related_context_enabled: bool` — default true. Whether to compute related_count/hint per recalled memory.

5. Nest all three new structs in `Config`:
   - `#[serde(default)] pub user: UserConfig`
   - `#[serde(default)] pub workspace: WorkspaceConfig`
   - `#[serde(default)] pub temporal: TemporalConfig`

Follow the exact pattern of existing configs: standalone `Default` impl, `#[serde(default)]` on each field with named default functions, `#[serde(default)]` on the Config-level field.

Add `Config::default()` entries for the three new sections.

# Phase 08.8 Plan 01: Plugin Support Primitives — Schema and Config Foundations Summary

**One-liner:** Database migration 018 adds event_time (TIMESTAMPTZ), event_time_precision (TEXT CHECK), and workspace (TEXT) columns with partial index; UserConfig, WorkspaceConfig, TemporalConfig config structs added; RecallConfig extended with truncation_chars, preamble_override, related_context_enabled; Memory and CreateMemory structs gain all three new fields with full store layer plumbing.

## What Was Built

### Task 1: Migration 018 + Config Structs

**Migration 018** (`crates/memcp-core/migrations/018_plugin_primitives.sql`):
- `ALTER TABLE memories ADD COLUMN IF NOT EXISTS event_time TIMESTAMPTZ` — nullable, stores when the content refers to (separate from created_at)
- `ALTER TABLE memories ADD COLUMN IF NOT EXISTS event_time_precision TEXT CHECK (...)` — "decade", "year", "month", "day" precision levels
- `ALTER TABLE memories ADD COLUMN IF NOT EXISTS workspace TEXT` — nullable, NULL = global scope
- `CREATE INDEX IF NOT EXISTS memories_workspace_idx ON memories (workspace) WHERE workspace IS NOT NULL` — partial index excludes global memories

**UserConfig** (`config.rs`):
- `birth_year: Option<u32>` — enables resolving "when I was 6" with config value

**WorkspaceConfig** (`config.rs`):
- `default_workspace: Option<String>` — default applied when no CLI flag or env var set

**TemporalConfig** (`config.rs`):
- `llm_enabled: bool` (default false) — gates background LLM temporal extraction worker
- `provider, ollama_model, openai_model, openai_api_key, openai_base_url` — provider config for when enabled

**RecallConfig extensions** (`config.rs`):
- `truncation_chars: usize` (default 200) — smart truncation for token efficiency
- `preamble_override: Option<String>` — custom session-start preamble
- `related_context_enabled: bool` (default true) — related_count/hint per recalled memory

All three new configs nested in root `Config` with `#[serde(default)]`.

### Task 2: Memory/CreateMemory Struct Updates + Store Layer Plumbing

**Memory struct** (`storage/store/mod.rs`):
- `event_time: Option<DateTime<Utc>>`
- `event_time_precision: Option<String>`
- `workspace: Option<String>`

**CreateMemory struct** (`storage/store/mod.rs`):
- Same three fields with `#[serde(default)]` — backward compatible JSON deserialization

**postgres.rs updates**:
- `row_to_memory()` updated with `try_get().unwrap_or(None)` for all three new columns

- store() return `Memory {}` construction includes new fields from CreateMemory input
- Curation query pair also updated (uses row_to_memory but needed SELECT column list updated)

**1. [Rule 3 - Blocking] CreateMemory struct literal initialization errors in 10 files**
- **Found during:** Task 2 (first build after struct change)

- **Fix:** Added `event_time: None, event_time_precision: None, workspace: None` to each CreateMemory struct literal
- **Files modified:** auto_store/mod.rs, curation/worker.rs, cli.rs, ingest.rs, server.rs, builders.rs, search_quality.rs, journey_test.rs

**2. [Rule 3 - Blocking] Curation query pair missing new columns**
- **Found during:** Task 2 (code review of all SELECT sites)
- **Issue:** get_memories_for_curation() had two query strings (with/without since) that selected parent_id/chunk_index/total_chunks but not the new columns — while row_to_memory would unwrap_or(None) gracefully, explicit column inclusion is better practice and consistent with all other queries
- **Fix:** Added event_time, event_time_precision, workspace to both query variants
- **Files modified:** postgres.rs
- **Commit:** 492d672

## Decisions Made

1. **event_time_precision as TEXT CHECK, not ENUM** — per CONTEXT.md/RESEARCH.md pitfall guidance, TEXT CHECK is easier to extend without ALTER TYPE
2. **Partial index on workspace WHERE workspace IS NOT NULL** — excludes global memories (NULL workspace) for index efficiency
3. **TemporalConfig.openai_base_url as Option<String>** — None means use the provider's default, not an explicit empty string; distinguishes absent-from-config vs set-to-default
4. **All 10 CreateMemory literals set new fields to None** — event_time population is reserved for Plan 03 (temporal extraction); workspace population for Plan 04

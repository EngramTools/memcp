2. Build the emission payload per CONTEXT.md:

```rust
    "memory_id": memory.id,
    "role": /* extract role from parsed entry metadata */,
    "tags": &memory.tags,
    "type_hint": &memory.type_hint,
    "content_preview": memory.content.chars().take(100).collect::<String>(),

});
```

3. Append to companion file (fail-open per Pitfall 4):
```rust
async fn append_id_emission(path: &Path, payload: &serde_json::Value) {
    let line = match serde_json::to_string(payload) {
        Ok(l) => l,
        Err(e) => {
            tracing::warn!(error = %e, "Failed to serialize ID emission");
            return;
        }
    };
    match tokio::fs::OpenOptions::new()
        .create(true)
        .append(true)
        .open(path)
        .await
    {
        Ok(mut file) => {
            use tokio::io::AsyncWriteExt;
            if let Err(e) = file.write_all(format!("{}\n", line).as_bytes()).await {
                tracing::warn!(error = %e, path = %path.display(), "Failed to write ID emission");
            }
        }
        Err(e) => {
            tracing::warn!(error = %e, path = %path.display(), "Failed to open companion file");
        }
    }
}
```

Key: Only emit for successfully stored memories. Filtered/rejected content is NOT emitted (the emission happens after the store call, which only runs if content passes filters).

**Part A: LLM temporal extraction function**

Add to `pipeline/temporal/mod.rs`:

```rust
/// LLM-based temporal extraction for subtle references regex can't catch.
/// Sends content to LLM with prompt asking for temporal references.
/// Returns extracted event_time + precision, or None if no temporal reference found.
pub async fn extract_event_time_llm(
    content: &str,
    birth_year: Option<u32>,
    provider: &str,  // "ollama" or "openai"
    model: &str,
    api_key: Option<&str>,
    base_url: Option<&str>,
) -> Option<(DateTime<Utc>, EventTimePrecision)>
```

Implementation:
1. Build a prompt:
```
Analyze this text and extract any temporal reference to when the described event happened.
If the person mentions their birth year context: they were born in {birth_year}.
Return ONLY a JSON object: {"year": YYYY, "month": MM_or_null, "day": DD_or_null} or "none" if no temporal reference.
Text: {content}
```

2. Call Ollama or OpenAI API (follow existing patterns in `summarization/ollama.rs` and `summarization/openai.rs` — reuse the HTTP client pattern, or better yet, call the existing LLM infrastructure if a shared client exists).

3. Parse the JSON response. Map to `(DateTime<Utc>, EventTimePrecision)` based on which fields are non-null.

4. Timeout: 3 seconds (matches LLM timeout pattern from Phase 08.5-03 LlmCategoryClassifier).

**Part B: Background worker**

Add a temporal LLM worker function:
```rust
pub async fn run_temporal_worker(
    store: Arc<PostgresMemoryStore>,

```

Implementation:
1. Poll for memories where `event_time IS NULL AND extraction_status = 'complete'` (already fully processed, just missing temporal data). Query: `SELECT id, content FROM memories WHERE event_time IS NULL AND deleted_at IS NULL ORDER BY created_at DESC LIMIT 10`.
2. For each memory, call `extract_event_time_llm(content, birth_year, ...)`.
3. If extraction succeeds, UPDATE the memory: `UPDATE memories SET event_time = $2, event_time_precision = $3 WHERE id = $1`.
4. Add a store method `update_event_time(id, event_time, precision)` to postgres.rs for this.
5. Poll interval: 60 seconds (low priority background work).
6. Gate on `config.temporal.llm_enabled` — when false, worker returns immediately.

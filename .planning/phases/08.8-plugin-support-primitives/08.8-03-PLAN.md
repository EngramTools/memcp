**Public API:**
```rust
/// Precision of the extracted event time
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub enum EventTimePrecision {
    Decade,
    Year,
    Month,
    Day,
}

impl EventTimePrecision {
    pub fn as_str(&self) -> &'static str {
        match self {
            Self::Decade => "decade",
            Self::Year => "year",
            Self::Month => "month",
            Self::Day => "day",
        }
    }
}

/// Extract event time from memory content using regex patterns.
/// Returns the most specific temporal reference found.
pub fn extract_event_time(
    content: &str,
    birth_year: Option<u32>,
    now: DateTime<Utc>,
) -> Option<(DateTime<Utc>, EventTimePrecision)>
```

**Regex patterns** — Use `std::sync::OnceLock` for static compilation per Pitfall 5:

1. **Absolute year**: `\bin\s+(\d{4})\b` — "in 2019", "in 1985". Validate year is 1900-2100. Precision: Year. Event time: Jan 1 of that year.

2. **Decade reference**: `\bin\s+the\s+(\d{2})s\b` — "in the 90s", "in the 80s". Precision: Decade. Event time: Jan 1 of decade start (e.g., 1990).

3. **Relative age**: `\bwhen\s+I\s+was\s+(\d{1,3})\b` — "when I was 6", "when I was 25". Requires `birth_year`. Precision: Year. Event time: Jan 1 of (birth_year + age). Skip if birth_year is None.

4. **Month-year**: `\bin\s+(January|February|March|April|May|June|July|August|September|October|November|December)\s+(\d{4})\b` — "in March 2020". Precision: Month. Event time: 1st of that month.

5. **Relative month**: `\blast\s+(January|February|March|April|May|June|July|August|September|October|November|December)\b` — "last March". Precision: Month. Event time: 1st of that month in the most recent past occurrence relative to `now`.

6. **Relative day**: `\blast\s+(Monday|Tuesday|Wednesday|Thursday|Friday|Saturday|Sunday)\b` — "last Tuesday". Precision: Day. Event time: most recent past occurrence relative to `now`.

**Priority:** Return the FIRST match found, scanning in pattern order (most specific first: month-year > year > decade > relative-age > relative-month > relative-day). If multiple matches, first wins — content typically has one primary temporal reference.

**Unit tests** — Add `#[cfg(test)] mod tests` at the bottom:
- "in 2019" → (2019-01-01, Year)
- "in the 90s" → (1990-01-01, Decade)
- "when I was 6" with birth_year=1990 → (1996-01-01, Year)
- "when I was 6" with birth_year=None → None
- "in March 2020" → (2020-03-01, Month)
- "last March" with now=2026-06-15 → (2026-03-01, Month)
- "last March" with now=2026-02-15 → (2025-03-01, Month)
- No temporal reference → None
- Content with multiple references → first match wins

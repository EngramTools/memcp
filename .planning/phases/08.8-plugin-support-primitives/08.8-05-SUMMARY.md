# Phase 08.8 Plan 05: Recall Output Improvements Summary

Enhanced recall output with session-start preamble injection (`--first`), smart content truncation, and related-context hints — giving agents token-efficient context injection with navigation affordances.

## What Was Built

### Task 1: Related context query + CLI helpers

**`get_related_context` in `postgres.rs`:**
- New `RelatedContext` struct: `related_count: i64` + `shared_tags: Vec<String>`
- Batch method fetches tags for all recalled memory IDs in one `SELECT ... WHERE id = ANY($1)` query
- Per-memory count uses `?|` (array overlap) SQL operator to count memories sharing any non-trivial tag
- Trivial tags filtered at query time: `auto-stored`, `summarized`, `merged`, `stale`, `curated:strengthened`, and any `category:*` prefix

**Helpers in `cli.rs`:**
- `truncate_content(content, max_chars) -> (String, bool)`: Unicode-aware truncation with `...` suffix
- `build_related_hint(shared_tags) -> String`: Builds `memcp search --tags tag1,tag2` command

### Task 2: `--first` flag + output assembly

**`main.rs`:**
- `Recall` command gains `--first: bool` arg with doc comment explaining session-start use case
- Match arm destructures `first` and passes to `cmd_recall`
- Fixed `AutoStoreWorker::spawn` call site (serve mode) for new `workspace`/`birth_year` params

**`cmd_recall` in `cli.rs`:**
- Signature: added `first: bool` parameter
- Fetches related context batch after recall (skipped when `config.recall.related_context_enabled = false`)
- Truncates each memory's content to `config.recall.truncation_chars` (default 200)
- Builds `memories` array with `id`, `content`, `relevance`, optional `truncated: true`, optional `related_count` + `hint`
- When `first=true`: injects `current_datetime` (RFC3339 Utc) and `preamble` into top-level output
- Preamble uses `config.recall.preamble_override` if set, else `DEFAULT_PREAMBLE` const
- Output when `first=false`: `{"session_id": "...", "count": N, "memories": [...]}`
- Output when `first=true`: `{"current_datetime": "...", "preamble": "...", "session_id": "...", "count": N, "memories": [...]}`

**Part A: Related context query in postgres.rs**

Add a store method to count memories sharing tags with a given memory:

```rust
/// For each memory ID, count how many other live memories share at least one
/// non-trivial tag, and return one representative set of shared tags for the hint.
pub async fn get_related_context(
    &self,
    memory_ids: &[String],
) -> Result<HashMap<String, RelatedContext>, MemcpError>
```

Where:
```rust
#[derive(Debug, Clone)]
pub struct RelatedContext {
    pub related_count: i64,
    pub shared_tags: Vec<String>,
}
```

SQL approach — for each memory:
1. Get the memory's tags (from the Memory struct, already available in recall results).
2. Query: count other memories that have at least one overlapping tag:
```sql
SELECT COUNT(DISTINCT m2.id) as related_count
FROM memories m2
WHERE m2.id != $1
  AND m2.deleted_at IS NULL
  AND m2.tags IS NOT NULL
  AND m2.tags ?| $2::text[]
```
where `$2` is the array of tags from the source memory.

3. For efficiency, batch all memory IDs in one call using a CTE or lateral join rather than N individual queries.

4. Filter out trivial tags from the hint — skip common tags like "auto-stored", "summarized". Consider filtering tags that start with "category:" or are in a short skip-list.

**Part B: Truncation helper in cli.rs**

Add a helper function:
```rust
fn truncate_content(content: &str, max_chars: usize) -> (String, bool) {
    if content.chars().count() <= max_chars {
        (content.to_string(), false)
    } else {
        let truncated: String = content.chars().take(max_chars).collect();
        (format!("{}...", truncated), true)
    }
}
```

**Part C: Build related context hint string**

In cli.rs, add a helper that builds the ready-made search command:
```rust
fn build_related_hint(shared_tags: &[String]) -> String {
    if shared_tags.is_empty() {
        return String::new();
    }
    format!("memcp search --tags {}", shared_tags.join(","))
}
```

**Part B: Update cmd_recall signature and implementation**

Update `cmd_recall` to accept `first: bool` parameter:
```rust
pub async fn cmd_recall(
    store: &Arc<PostgresMemoryStore>,
    config: &Config,
    query: &str,
    session_id: Option<String>,
    reset: bool,
    first: bool,
    workspace: Option<String>,  // from Plan 03's --workspace addition
) -> Result<(), MemcpError>
```

Implementation changes:

1. **Preamble (when first=true):**
   Define a hardcoded default preamble:
   ```rust
   const DEFAULT_PREAMBLE: &str = "You have access to persistent memory via memcp. \
   Key commands: `memcp store \"content\" --tags tag1,tag2` to save, \
   `memcp search \"query\"` to find, `memcp get <id>` for full content, \
   `memcp annotate --id <id> --tags tag1 --salience 1.5x` to enrich. \
   Memories persist across sessions. Store important decisions, preferences, and context.";
   ```
   Use `config.recall.preamble_override` if set, otherwise use `DEFAULT_PREAMBLE`.

2. **Output assembly:**
   Build the output JSON object:
   ```rust
   let mut output = json!({});

   if first {
       output["current_datetime"] = json!(Utc::now().to_rfc3339());
       output["preamble"] = json!(preamble_text);
   }
   ```

3. **Truncation:**
   For each recalled memory, apply truncation:
   ```rust
   let truncation_chars = config.recall.truncation_chars;
   let (truncated_content, was_truncated) = truncate_content(&memory.content, truncation_chars);
   ```
   Build memory output object with truncated content. Include `"truncated": true` flag when content was truncated so the agent knows to `memcp get <id>` for full content.

4. **Related context:**
   If `config.recall.related_context_enabled`:
   - Collect all recalled memory IDs.
   - Call `store.get_related_context(&memory_ids).await?`
   - For each memory in output, add:
   ```json
   {
     "related_count": 5,
     "hint": "memcp search --tags decision,project-x"
   }
   ```
   - Only include related_count/hint when related_count > 0 (skip noise for isolated memories).

5. **Memory output format:**
   Each memory in the `memories` array:
   ```json
   {
     "id": "abc-123",
     "content": "Truncated content here...",
     "truncated": true,
     "type_hint": "fact",
     "tags": ["tag1", "tag2"],
     "event_time": "2019-01-01T00:00:00Z",
     "workspace": "myproject",
     "related_count": 3,
     "hint": "memcp search --tags tag1,tag2"
   }
   ```

6. **Final output:**
   Print the assembled JSON to stdout. When first=false: `{"memories": [...]}`. When first=true: `{"current_datetime": "...", "preamble": "...", "memories": [...]}`.

```rust
pub struct SimilarMemory {
    pub memory_id: String,
    pub similarity: f64,
    pub content: String,
}

pub async fn find_similar_memories(
    pool: &sqlx::PgPool,
    memory_id: &str,
    embedding: &pgvector::Vector,
    threshold: f64,
    limit: i64,
) -> Result<Vec<SimilarMemory>, MemcpError>;
```

The function queries:
- memory_embeddings table (me.embedding, me.is_current, me.memory_id)
- memories table (m.content, m.embedding_status, m.is_consolidated_original, m.deleted_at)
- Uses cosine distance: `1 - (me.embedding <=> $1)` as similarity score
- Excludes: source memory, consolidated originals, deleted, non-complete embedding status

**Setup pattern for each test:**
1. Use `#[sqlx::test(migrator = "memcp::MIGRATOR")]` with `pool: PgPool`
2. Create a `PostgresMemoryStore::from_pool(pool.clone())`
3. Store 2-3 memories via `store.store(...)`
4. Manually insert embeddings into `memory_embeddings` table using raw SQL since the embedding pipeline requires fastembed. Use `pgvector::Vector::from(vec![...])` with 384-dimensional vectors.
5. For similar vectors, use nearly identical vectors (e.g., same values with small perturbation on 1-2 dimensions).
6. For dissimilar vectors, use orthogonal vectors.
7. Call `find_similar_memories()` and assert results.

**Embedding insertion SQL:**
```sql
INSERT INTO memory_embeddings (memory_id, embedding, model_name, is_current)
VALUES ($1, $2, 'test', TRUE)
```

Also update `memories.embedding_status = 'complete'` after inserting embeddings.

**Test implementations:**

`test_find_similar_above_threshold`:
- Store memory A and memory B with nearly identical 384-dim embeddings (cosine sim > 0.95)
- Insert embeddings manually, set embedding_status = 'complete'
- Call find_similar_memories(pool, A.id, A_embedding, 0.8, 10)
- Assert returns 1 result with B's id, similarity > 0.8

`test_find_similar_excludes_self`:

- Call find_similar_memories
- Assert the consolidated original is excluded

`test_find_similar_respects_limit`:
- Store 3 memories with very similar embeddings
- Call with limit=1
- Assert returns exactly 1 result

**Vector construction helper:**
```rust
fn make_base_vector() -> Vec<f32> {
    let mut v = vec![0.0f32; 384];
    // Set first 10 dimensions to create a recognizable pattern
    for i in 0..10 { v[i] = 1.0 / (i as f32 + 1.0); }
    v
}

fn make_similar_vector() -> Vec<f32> {
    let mut v = make_base_vector();
    v[0] += 0.01; // tiny perturbation
    v
}

fn make_orthogonal_vector() -> Vec<f32> {
    let mut v = vec![0.0f32; 384];
    for i in 10..20 { v[i] = 1.0 / (i as f32 + 1.0); }
    v
}
```

Note: pgvector cosine distance requires non-zero vectors. The helper functions produce non-zero vectors by construction.

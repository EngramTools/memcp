## Accomplishments

- `client.rs`: `run_workload()` spawns N tasks bounded by `Arc<Semaphore>` with concurrency permits, collects `RequestResult` per request via `Arc<Mutex<Vec>>`, shows progress with indicatif

- `bin/load_test.rs`: Full CLI with 12 flags, DB connect, migration, corpus seed, axum test server spawn, paired raw/rate-limited workload runs, JSON+Markdown report save, baseline comparison, Fly.io tier output

## Decisions Made

- Recall uses `first: true` (queryless) instead of query-based recall to avoid requiring an `embed_provider` in the load test binary (which uses no embedding). This still exercises the recall endpoint.
- Write cycle falls back to `store` when no stored IDs exist (start of run, or after many deletes) rather than skipping, ensuring write_pct is always honoured.
- Non-global Prometheus recorder (`PrometheusBuilder::new().build_recorder()`) avoids panic when multiple runs are executed in the same process.
- `futures` crate not added — handle collection uses a simple `for handle in handles { let _ = handle.await }` loop since tokio's JoinHandle awaits sequentially.

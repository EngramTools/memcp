# Phase 08.11 Plan 02: Warm Recall Session Ranking Summary

**End-to-end tag-affinity boosting: recall() and recall_queryless() apply explicit + session tag boost after scoring, CLI gains --boost-tags flag, MCP gains boost_tags param, session tags accumulate automatically on every recall**

## Accomplishments

- `recall_candidates()` extended to SELECT `m.tags` — returns `Vec<(String, String, f32, Option<serde_json::Value>)>`. Both SQL tiers (extraction-on and extraction-off) include tags. `recall_candidates_multi_tier` updated to match new return type.
- `RecallEngine::recall()` gains `boost_tags: &[String]` param. After candidate fetch: fetches session tags (if session_topic_tracking enabled), computes explicit + session boost per candidate via `compute_tag_boost`, builds `RecalledMemory` with boosted relevance + boost metadata, re-sorts by boosted relevance, accumulates candidate tags into session via `accumulate_session_tags`.
- `RecallEngine::recall_queryless()` gains `boost_tags: &[String]` param. After `SalienceScorer::rank()`: fetches session tags, applies boost additively to `hit.salience_score`, re-sorts hits by boosted score, truncates, builds `RecalledMemory` with boost fields, accumulates session tags.
- CLI `Commands::Recall` gains `--boost-tags` with `value_delimiter=','`. `cmd_recall` gains `boost_tags: &[String]` param threaded to both recall engine paths. CLI JSON output includes `boost_applied`/`boost_score` when non-zero.
- MCP `RecallMemoryParams` gains `boost_tags: Option<Vec<String>>`. Handler extracts with `unwrap_or_default()` and passes to both query-based and queryless engine paths.
- All existing call sites in test files updated with `&[]` for backward compatibility.

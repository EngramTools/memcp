## Implementation Notes

### CLI Integration
- `--profile trust` routes to `run_trust_workload_cli()` before standard profile parsing
- `--real-llm` flag controls mock vs Ollama LLM latency (0ms real, 200ms mock)
- Standard profile flow is unchanged (profile defaults to "standard")

### Trust Workload Orchestrator
Full pipeline in `run_trust_workload_cli`:
1. Clear corpus + curation_runs tables
2. Seed trust corpus via `seed_trust_corpus()`
3. Initialize TrustWorkloadState with poisoned IDs
4. Spawn curation loop (5s interval, sequential lock prevents overlap)
5. Spawn HTTP workload concurrently (reuses existing client.rs)
6. Signal shutdown, wait for curation to stop
7. Run post_run_audit against DB
8. Build SecurityReport from accumulated metrics
9. Generate combined markdown report (standard + security section)

### Post-Run Audit
- Queries all poisoned memory IDs from DB
- For quarantined memories: runs `audit_memory()` for tag/trust/audit consistency
- Updates quarantined_ids set for accurate detection rate calculation
- Does NOT flag undetected poisons (detection depends on trust-gated thresholds)

### Security Report Section
Renders markdown with:
- Detection summary table (seeded, quarantined, rate, false positives, violations)
- Violations list (if any) or "No violations found"
- Curation throughput: P1/P2/Normal drain time stats
- Poison dwell time: mean and p95 (store-to-quarantine latency)

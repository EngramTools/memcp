## Decisions Made
- NULL-safe suspicious tag exclusion using `(tags IS NULL OR NOT (tags @> '["suspicious"]'::jsonb))` instead of bare `NOT` to handle memories with NULL tags column
- Used `std::sync::LazyLock` (stable since Rust 1.80) instead of `once_cell::sync::Lazy` for regex compilation
- Signal deduplication in detect_injection_signals to avoid double-counting from multiple regex patterns matching the same signal name

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] NULL tags exclusion fix**
- **Found during:** Task 1 (search exclusion tests)
- **Issue:** `NOT (tags @> '["suspicious"]'::jsonb)` returns NULL (not true) when tags column is NULL, inadvertently excluding all memories without tags from search results
- **Fix:** Changed to `(tags IS NULL OR NOT (tags @> '["suspicious"]'::jsonb))`
- **Files modified:** crates/memcp-core/src/storage/store/postgres.rs
- **Verification:** All 5 integration tests pass with both tagged and untagged memories

**2. [Rule 3 - Blocking] Fixed store_and_embed helper for memory_embeddings schema**
- **Found during:** Task 1 (integration tests)
- **Issue:** Shared test helper used `ON CONFLICT (memory_id)` but memory_embeddings table has `id` as PK, not a unique constraint on memory_id
- **Fix:** Created test-local `store_and_embed_for_test` using direct INSERT with UUID id
- **Files modified:** crates/memcp-core/tests/curation_security_test.rs
- **Verification:** All integration tests pass

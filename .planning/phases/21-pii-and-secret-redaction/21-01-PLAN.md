  <action>
    1. Create `crates/memcp-core/src/pipeline/redaction/` directory with three files:

    **entropy.rs**: Shannon entropy calculation (~15 lines). `pub fn shannon_entropy(s: &str) -> f64`. Byte-frequency based, returns bits per character. 0.0 for empty strings.

    **rules.rs**: Default pattern definitions.
    - `pub struct RedactionRule` with fields: `category: String`, `regex: Regex`, `screening_pattern: String` (for RegexSet), `mask_style: MaskStyle`, `entropy_check: bool`, `rule_type: RuleType`.
    - `pub enum MaskStyle { Partial { prefix_len: usize }, Full }`
    - `pub enum RuleType { Secret, Pii }`
    - `pub fn default_secret_rules() -> Vec<RedactionRule>` — 13 patterns from RESEARCH.md (AWS, Anthropic, OpenAI, GitHub PAT, GitHub fine-grained, Slack, Stripe, npm, GCP, Twilio, HuggingFace, PyPI, generic secret).
    - `pub fn default_pii_rules() -> Vec<RedactionRule>` — SSN and credit card only (per user decision: no email, no phone, no IP).
    - Helper constructors: `RedactionRule::secret(category, pattern, mask_style, entropy_check)` and `RedactionRule::pii(...)`.

    **mod.rs**: RedactionEngine.
    - `pub struct RedactionResult { pub content: String, pub redaction_count: usize, pub categories: Vec<String>, pub was_redacted: bool }` with `clean(content)` constructor.
    - `pub struct RedactionEngine { screening_set: RegexSet, rules: Vec<RedactionRule>, allowlist_values: HashSet<String>, allowlist_patterns: Option<RegexSet> }`.
    - `pub fn from_config(config: &RedactionConfig) -> Result<Self, MemcpError>`:
      - Collect rules based on config (secrets if `secrets_enabled`, PII if `pii_enabled`).
      - Add custom rules from config.
      - Build RegexSet from screening patterns (same order as rules vec).
      - Build allowlist structures.
    - `pub fn redact(&self, content: &str) -> Result<RedactionResult, MemcpError>`:
      - Phase 1: `screening_set.matches(content)` — if empty, return `RedactionResult::clean(content)`.
      - Phase 2: For each matched rule index, find all matches with the individual regex. For each match:
        - Check allowlist (exact value match or pattern match) — skip if allowlisted.
        - If rule.entropy_check, compute shannon_entropy on captured group — skip if below config threshold.
        - Apply masking: Partial → preserve first N chars + `[REDACTED:category]`, Full → `[REDACTED:category]`.
      - Process replacements right-to-left (by byte offset) to avoid offset shifting.
      - Collect unique categories, count total redactions.
    - Important: Do NOT match inside existing `[REDACTED:...]` markers (skip matches where captured text starts with `[REDACTED:`).

    ```rust
        #[serde(default)]
        pub allowlist: AllowlistConfig,
        #[serde(default)]
        pub custom_rules: Vec<CustomRuleConfig>,
    }
    ```

    Default functions: `default_secrets_enabled() -> true`, `default_entropy_threshold() -> 3.5`.

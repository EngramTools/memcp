**Step 1: Rename HealthState to AppState in transport/health/mod.rs**

Replace `HealthState` with `AppState`. Add new fields needed by API handlers:

```rust
#[derive(Clone)]
pub struct AppState {
    // Existing fields:
    pub ready: Arc<AtomicBool>,
    pub started_at: Instant,
    pub caps: ResourceCapsConfig,
    pub store: Option<Arc<PostgresMemoryStore>>,
    // New fields for API:
    pub config: Arc<Config>,
    pub embed_provider: Option<Arc<dyn EmbeddingProvider + Send + Sync>>,
    pub embed_sender: Option<mpsc::Sender<EmbeddingJob>>,
}
```

- `config: Arc<Config>` â€” needed by recall handler (RecallConfig, ExtractionConfig), store handler (resource caps, temporal extraction), search handler.
- `embed_provider: Option<Arc<dyn EmbeddingProvider + Send + Sync>>` â€” recall handler needs to embed queries in-process (no IPC). Use the router's `default_provider()`.
- `embed_sender: Option<mpsc::Sender<EmbeddingJob>>` â€” store handler needs to enqueue embedding jobs for `wait: true` sync store pattern (oneshot completion channel on EmbeddingJob).
- `EmbeddingProvider` is `Send + Sync` but NOT `Clone` directly. Wrap in `Arc<dyn ...>` which IS `Clone`. This keeps `AppState: Clone` (required by axum `with_state`).

**Handler pattern (all handlers follow this):**
```rust
pub async fn handler(
    State(state): State<AppState>,
    Json(req): Json<RequestType>,
) -> (StatusCode, Json<serde_json::Value>) {
    // 1. Check readiness
    if !state.ready.load(Ordering::Acquire) {
        return (StatusCode::SERVICE_UNAVAILABLE, Json(error_json("daemon not ready")));
    }
    // 2. Get store
    let store = match &state.store {
        Some(s) => s.clone(),
        None => return (StatusCode::SERVICE_UNAVAILABLE, Json(error_json("store not available"))),
    };
    // 3. Validate required fields (return 400 if missing)
    // 4. Call store/engine methods directly
    // 5. Return (StatusCode::OK, Json(result))
}
```

**recall.rs â€” POST /v1/recall**

1. Extract `query` from request. If `query` is None or empty string, use queryless path.
2. Construct `RecallEngine::new(store, config.recall.clone(), config.extraction.enabled)`.
3. For query-based: embed query using `state.embed_provider` directly (in-process, NOT embed_via_daemon IPC). Call `provider.embed(query).await`. If embed_provider is None, return 503.
4. Call `engine.recall()` or `engine.recall_queryless()`.
5. For query-based with `first=true`, fetch project summary via `store.fetch_project_summary()`.
6. Fetch related context via `store.get_related_context()` if `config.recall.related_context_enabled`.
7. Build JSON response matching CLI output shape: `{ session_id, count, memories: [...], summary?, current_datetime?, preamble? }`.
8. Apply truncation using `config.recall.truncation_chars`. Include `truncated: true` field when content was truncated.
9. Include boost metadata when `boost_applied=true`. Include related context (related_count, hint) when present and count > 0.
10. Include `current_datetime` and `preamble` when `first=true` (same as CLI).

Note: The recall handler replicates the data-building logic from `cmd_recall` (lines 1407-1510 in cli.rs). This is intentional per CONTEXT.md â€” HTTP handlers bypass cli.rs. The logic is straightforward JSON assembly, not complex business logic. Duplicating it avoids coupling the API to CLI internals. Use `crate::cli::truncate_content()` and `crate::cli::build_related_hint()` if they are pub, otherwise duplicate the trivial truncation/hint logic.

**search.rs â€” POST /v1/search**

1. Validate `query` is non-empty (400 if empty).
2. Build `SearchFilter` from request fields (query, limit, tags, source, type_hint, audience, workspace, min_salience, cursor).
3. Embed query via `state.embed_provider`. If None, perform text-only search (BM25 only â€” set `query_embedding` to empty/placeholder).
4. Call `store.hybrid_search()` or the appropriate multi-tier search method.
5. Apply field projection if `fields` param is set (reuse `crate::cli::apply_field_projection` if pub, or inline).
6. Return JSON array of results matching CLI `--json` format.

Note: Search is more complex than other handlers because of the full hybrid pipeline. The CLI path uses `embed_via_daemon()` + salience re-ranking + QI reranking. The HTTP handler has access to `embed_provider` but not the full QI reranking pipeline. For this phase: embed via provider, run hybrid search, apply salience scoring. QI reranking is a nice-to-have and can be added later. The response shape must match CLI `--json` output.

**store.rs â€” POST /v1/store**

1. Validate `content` is non-empty (400 if empty).
2. Resource cap check: replicate the logic from `cmd_store` lines 251-266. Check `config.resource_caps.max_memories` against `store.count_live_memories()`. Return 400 if hard cap exceeded. Include warning header if at warn threshold.
3. Extract temporal event time from content: call `crate::cli::extract_event_time()` (already pub). Pass `config.user.birth_year` and `Utc::now()`.
4. Build `CreateMemory` struct from request fields + temporal extraction results.
5. Call `store.store(input).await`.
6. Seed salience: `store.upsert_salience(&memory.id, 3.0, 5.0, 0, None)`.
7. If `wait: true`: use the oneshot channel pattern from Phase 08.5. Create a `tokio::sync::oneshot::channel()`. Build an `EmbeddingJob` with `completion_tx: Some(tx)` and send via `state.embed_sender`. Await the receiver with `config.store.sync_timeout_secs` timeout. On completion, fetch updated memory to get final embedding_status.
   - If `embed_sender` is None and `wait: true`, fall back to polling (like CLI does) or just return pending status.
   - If `wait: false` (default): return immediately with `"embedding_status": "pending"`.
8. Return `{ "id": memory.id, "embedding_status": "..." }` plus other memory fields matching `format_memory_json` output.

**annotate.rs â€” POST /v1/annotate**

1. Validate `id` is non-empty (400 if empty).
2. Call `crate::cli::annotate_logic(&store, &req.id, req.tags, req.replace_tags, req.salience)`. This shared helper already exists.
3. On success: return the diff JSON `{ tags_added, tags_removed, salience_before, salience_after }`.
4. On error (memory not found): return 404.

**update.rs â€” POST /v1/update**

1. Validate `id` is non-empty (400 if empty).
2. Validate at least one of content, type_hint, source, tags is Some (400 if all None).
3. Build `UpdateMemory` struct from request fields.
4. Call `store.update(&req.id, input).await`.
5. If content changed, reset embedding_status to pending: `store.reset_embedding_status(&req.id).await` (or however cmd_update does it â€” check cli.rs lines 346+).
6. Return the updated memory JSON.
7. On error (not found): return 404.
